# AI 에이전트 팀 + 자기학습 일지 (프로토타입)

유튜브 영상에서 본 두 아이디어를 합친 최소 구현입니다.

1. **에이전트 팀** — 역할을 나눈 에이전트(기획자 → 실행가 → 검토자)가 한 작업을 이어받아 협업.
2. **자기학습 일지** — 각 에이전트가 일을 마치면 "무엇을 했고/통했고/막혔고/다음엔 이렇게"를 일기로 남기고, 다음 작업 전에 관련 일기를 회상해 적용 → **스스로 지혜를 쌓는 구조**.

영상의 "인간의 역할(최종 의사결정)"도 검토자가 `🧑 사람 결정 필요` 로 표시해 가볍게 반영했습니다.

## 빠르게 보기

```bash
python -m agent_team.demo
```

같은 작업을 두 번 돌립니다. **1차**엔 참고할 과거 일기가 없지만, **2차**엔 1차에서 남긴 일기를 회상해 더 많은 교훈을 적용합니다. 마지막에 누적된 지혜가 출력됩니다.

```
=== 1차 실행 ===
   적용한 과거 교훈: 0건
=== 2차 실행 (1차 일기를 회상) ===
   적용한 과거 교훈: 9건   ← 1차의 일기 3건을 세 에이전트가 각각 회상
```

## 구조

```
agent_team/
  llm.py      # LLM 추상화 + 키 없이 도는 MockLLM (+ get_llm 교체 지점)
  journal.py  # 자기학습 일지: 기록(record)·회상(recall)·지혜(wisdom)
  agents.py   # 기획자/실행가/검토자 역할 에이전트
  team.py     # 오케스트레이션 + 결과 렌더링 + 사람 결정 게이트
  demo.py     # 학습 루프 데모
```

핵심 흐름:

```
작업 ─▶ [기획자]──일기─┐
          │           │
          ▼           ▼
       [실행가]──일기─┤   회상(recall): 새 작업 전, 태그가 겹치는 과거 일기를 꺼내 프롬프트에 주입
          │           │
          ▼           ▼
       [검토자]──일기─┘──▶ 🧑 사람 최종 결정
```

## LLM 백엔드

지금은 **MockLLM**(키 불필요, 결정적)으로 동작합니다 — 구조·테스트 검증용이라 실제 추론은 하지 않습니다.

진짜 Claude 로 바꾸려면 `ANTHROPIC_API_KEY` 를 넣고 `llm.py` 의 `get_llm()` 한 곳에서 Anthropic 어댑터를 돌려주도록 확장하면 됩니다(그때 prompt caching 까지 갖춰 붙일 예정). 나머지 로직(일지·회상·팀·사람 게이트)은 그대로 재사용됩니다.
